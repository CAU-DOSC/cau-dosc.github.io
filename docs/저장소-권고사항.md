GitHub 저장소 운영 관련

- 오픈소스 라이선스 명시 (LICENSE 파일 및 소스코드 상단 주석)
- README.md 파일작성
  - 프로젝트 소개
  - 설치 및 실행 방법
  - 프로젝트에 기여하는 방법
- Issues/Pull requests/Projects 탭을 활용하여 프로젝트를 관리하거나 GitHub 외부 툴을 사용한다면 README.md에 명시
- git commit의 품질 관리
  - 하나의 commit에는 가급적 하나의 변경점만 포함하도록 작성
  - commit 메시지는 해당 commit 변경사항을 누락없이 요약해야 함
- 소스코드 내 주석 관리
